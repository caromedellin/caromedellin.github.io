Enumerable Methods

Map

Allows us to iterate over every method and it changes it and creates another array

This piece will grab every element of the array and convert it to a string

arr.map {|x| x.to_s}

Ruby also supports this syntax:

arr.map(&:to_s)

This is useful to debugging later, you can use t with Booleans too!

arr.map(&:odd?)

The difference on map and .each, each calls a block once for each element we iterate over and then it passes that element as a parameter. Map does the same but it also creates a new array! Once is finished looping we have a new array with map! makes it unnecessary to add the values created to a new temporary array. We just need to assign the array.map to a variable

So we use map new need a new array returned as well, we can use each when we don't necessarily need a new array returned.

      array = ["Puppie", "Dog", "Raccoon"]
      counter = 1
      array.each do |animal|
        puts "Prints the counter as a string #{counter}"
        puts "Animals I need in my life #{animal}!"
        counter += 2
      end

This block will not print the counter to be 1,3 and then end because it's not iterating over the index of the array it will print the counter to be 1,2,5 because it's iterating over the number of elements in the array
